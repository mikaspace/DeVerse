During this activity, I observed that sequential algorithms execute tasks one by one, while parallel algorithms run multiple parts at the same time. For small datasets,
the sequential version was faster because it has less overhead. However, for large datasets, the parallel version performed better since it used multiple processes.
I did find find some trouble managing the processes and making sure the results were combined correctly, especially in sorting where merging is required.
I also learned that parallel algorithms introduce extra overhead like process creation and communication.
